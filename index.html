New BIM 360 API (model coordination)

§  The BIM 360 Model Coordination API provides full access to the set of services used by the BIM 360 Model Coordination web application. It enables users to detect and manage the issues which arise when 3D models from different design disciplines are combined into a unified project coordination space.

§  Create and view coordination space model sets and versions.

§  Create virtual model set views by combining user defined views extracted from design files uploaded to BIM 360 Document Management.

§  Run Navisworks-like clash tests against model set versions.

§  Group clashes and assign their resolution to project users via the BIM 360 Issues Service.

§  Permanently ignore clashes which are expected or are part of the design.

§  Run SQL-like breakdown queries against a unified coordination space BIM property index.

§  For more information on BIM 360 Model Coordination product features, see BIM 360 Model Coordination product help documentation.

§  In beta: https://forge.autodesk.com/en/docs/bim360/v1/overview/introduction/

§  Xiaodong, Mikako are the experts here, if someone needs deeper understanding.

Forge demos

Viewer Rocks: https://viewer-rocks.autodesk.io/ 
Custom viewer site. headless viewer with custom UI and controls to drive the viewer. Gallery of pre-translated models.

Product Design/Asset Review/M&E: http://giro-watch.tk/ 
Using THREE js directly able to provide transforms / specialized kinematics to test joints etc. Requires extra programming, but for a "family" of assets could be possible to setup previews. Clock was authored in Autodesk Fusion 360.

MFG Digital Twin: https://forge-digital-twin.autodesk.io/ 
Good viewing large model from Inventor. Can show complex Model browser data and explain parts and assembly structure. Many parts and assemblies have properties from metadata. Explode looks great. Measure is also easy to demo here. There are three extensions (custom functionality, including UI elements on right): 1. Heat Map shows data on the parts. See popup panel that allows to select heatmap data and change intensity. 2. Issues / Markup. Allows to view issues. Looks like you can add new issues, but getting crowded. You could show how issues are linked to parts though. Click Maintenance tab, then click Isues sub-tab. Then you can click on a part number in the issues list and viewer shows it to you. 3. Real time data Animation. This one need to explain that we are overriding default Viewer behavior with THREE.js library to provide the animation.

Industrial Digital Twin Demo: https://forge-industrial-construction.herokuapp.com/
Same idea as above MFG with industrial data set

Point Clouds Demo: https://forge-point-clouds.herokuapp.com/

AR/VR: http://forgetoolkit.com/#/ 
This is a tookit that post-processes the Forge Viewer format into data that can be pulled into Unity. Works in two parts... After using Model Derivative to translated design to Forge Viewer format (SVF), you can then send it to AR VR toolkit services to generate better suited model for Unity and AV VR applications (but not limited). Via a Unity plugin, you can then statically import the asset, or dynamically load it from the server at runtime. This gives advantage of always having latest model in the AR VR experience. Limitations are device based.


IoT: https://forge-rcdb.autodesk.io/configurator?id=58adee163e6f342cf1e92dae 
Click on a hotspot in the model to show 'streaming IoT data'. Or click on an item in the list on the right. The data is fake - its just a demo. There is a way to connect your laptop to a real sensor and stream the data live to the sample, creating a much more impressive user experience. You need a texas sensor tag and running that project locally, plus a bluetooth connection: https://github.com/leefsmp/sensor-broker


AEC: https://forge-rcdb.autodesk.io/configurator?id=57f3739777c879f48ad54a44 
Visual Reports
Multiple Viewers on same page. 3D and 2D. Can change 2D view. Linked geometry data between viewers. Meta data used to build reports using 3rd party JS graph library. Data linked back to geomtry (ie. select a graphing element and show how selection is made in viewer).

BIM360 Issue Editor: https://bim360-issue-editor.herokuapp.com/


MFG: https://forge-rcdb.autodesk.io/database?id=57efaead77c8eb0a560ef465 
Database/Metadata
The material list on the top right is a display from an online database. Click a material to see every object made from that material. Open the Properties for an element in the model, scroll down to see the 'Database' properties. These have been injected into the Properties window from the database. Edit the material price in the database and the price in the property window changes too.
Careful with extensions.... I had some trouble with them recently...
